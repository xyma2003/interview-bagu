# 面试备战学习路线 & 刷题顺序

> 针对字节 / 腾讯 / 小红书 / 阿里 / MiniMax 等互联网大厂
> 适合：有一定编程基础，准备 3 个月内面试的同学

---

## 一、总体节奏建议

| 时间 | 阶段 | 目标 |
|------|------|------|
| 第 1-2 周 | 语言基础 + 数据结构入门 | 能独立写对简单 LeetCode |
| 第 3-5 周 | 算法核心专题 | 中等题目思路清晰、代码干净 |
| 第 6-8 周 | 八股文（系统知识） | 每个知识点能说清楚 + 有项目经验联系 |
| 第 9-10 周 | 系统设计 + 大厂真题 | 能独立设计中等规模系统 |
| 第 11-12 周 | 模拟面试 + 查漏补缺 | 高频题手写无 bug，八股流利表达 |

**每日目标**：算法 1-2 题（精做，写题解）+ 八股 3-5 个知识点（理解，能复述）

## 二、算法刷题顺序

### Phase 1：基础数据结构（第 1-2 周）

按以下顺序刷，每类先做 Easy 再做 Medium，**不要一开始就做 Hard**：

**Step 1：数组 + 哈希表（3 天）**
```
两数之和 (LC 1) → 字母异位词分组 (LC 49) → 前 K 高频元素 (LC 347)
→ 最长连续序列 (LC 128) → 除自身以外的乘积 (LC 238)
```
- 目标：学会"空间换时间"，哈希表是数组题的万能工具

**Step 2：双指针 + 滑动窗口（3 天）**
```
移动零 (LC 283) → 盛最多水 (LC 11) → 三数之和 (LC 15)
→ 最长无重复子串 (LC 3) → 最小覆盖子串 (LC 76)
```
- 目标：掌握"左右指针夹逼"和"快慢指针"两种模式

**Step 3：栈 + 队列（2 天）**
```
有效括号 (LC 20) → 最小栈 (LC 155) → 每日温度 (LC 739)
→ 柱状图最大矩形 (LC 84) → 滑动窗口最大值 (LC 239)
```
- 目标：掌握单调栈（下一个更大/更小元素类问题）

**Step 4：链表（2 天）**
```
反转链表 (LC 206) → 合并有序链表 (LC 21) → 环检测 (LC 141/142)
→ 相交链表 (LC 160) → 删除倒数第 N 个 (LC 19)
→ K 个一组翻转 (LC 25)
```
- 目标：哑节点 + 快慢指针是链表题的核心工具

---

### Phase 2：树与图（第 3-4 周）

**Step 5：二叉树（3 天）**
```
层序遍历 (LC 102) → 最大深度 (LC 104) → 前序/中序构造树 (LC 105)
→ 验证 BST (LC 98) → LCA (LC 236) → 最大路径和 (LC 124)
→ 右视图 (LC 199) → 展开为链表 (LC 114)
```
- **关键洞察**：绝大多数二叉树题 = 递归定义清晰的返回值

**Step 6：BFS / 图（2 天）**
```
岛屿数量 (LC 200) → 腐烂橘子 (LC 994) → 克隆图 (LC 133)
→ 课程表 I/II (LC 207/210) → 单词接龙 (LC 127)
```
- **关键洞察**：BFS = 队列；DFS = 递归或栈；拓扑排序 = BFS + 入度

**Step 7：回溯（3 天）**
```
全排列 (LC 46) → 子集 (LC 78) → 组合总和 (LC 39)
→ 单词搜索 (LC 79) → N 皇后 (LC 51)
```
- **统一模板**：`for 选择 in 选择列表: 做选择 → 递归 → 撤销选择`

---

### Phase 3：动态规划（第 5 周）

**刷题顺序（由易到难）**：
```
爬楼梯 (LC 70) → 打家劫舍 (LC 198) → 最大子数组和 (LC 53)
→ 跳跃游戏 (LC 55) → 零钱兑换 (LC 322) → 完全平方数 (LC 279)
→ 最长递增子序列 (LC 300) → 最长公共子序列 (LC 1143)
→ 编辑距离 (LC 72) → 分割等和子集 (LC 416)
→ 买卖股票系列 (LC 121/122/123) → 单词拆分 (LC 139)
→ 打家劫舍 III-树形 DP (LC 337) → 最大正方形 (LC 221)
```

**DP 学习方法**：
1. 先想状态定义（dp[i] 表示什么）
2. 写状态转移方程
3. 确定初始值和遍历顺序
4. 先暴力递归 + memo，再改成自底向上

---

### Phase 4：二分查找 + 高级数据结构（第 6 周）

```
二分查找模板 (LC 704/35/34) → 搜索旋转数组 (LC 33) → 寻找中位数 (LC 4)
→ 前 K 大元素 (LC 215/347) → 合并 K 个链表 (LC 23)
→ 实现 Trie (LC 208) → LRU Cache (LC 146)
```

---

### 算法刷题策略

**每道题的处理步骤**：
1. 独立思考 15-20 分钟
2. 写出思路（不是直接写代码）
3. 实现代码，测试边界
4. 看最优解，对比自己的方案
5. **第 2 天不看答案重写一遍**（间隔重复是记忆关键）

**不要做的事**：
- ❌ 没思路直接看答案（失去锻炼机会）
- ❌ 只做 Easy（Hard 才是面试主战场）
- ❌ 刷题不总结模板（每类题有固定套路）
- ❌ 把所有时间放算法，忽视八股（两者都要）

## 四、系统设计学习顺序（第 9-10 周）

### 先掌握基础组件

每个组件搞清楚：**是什么 → 核心原理 → 适用场景 → 注意事项**

```
Level 1（必须）：
  Load Balancer → Cache (Redis) → Message Queue (Kafka)
  → Database (MySQL 主从 + 分库分表) → CDN

Level 2（重要）：
  分布式 ID (Snowflake) → 分布式锁 → 限流 (令牌桶)
  → 熔断器 → 一致性哈希

Level 3（加分）：
  Service Mesh → 分布式事务 (Saga) → 向量数据库
```

### 经典系统设计题目（按难度）

**初级（2 周内搞定）**：
- 设计短链接系统（URL Shortener）
- 设计新闻订阅推送系统（News Feed）
- 设计排行榜（Leaderboard）

**中级（重点练习）**：
- 设计消息系统（IM，支持千万在线）
- 设计推荐系统（小红书/抖音场景）
- 设计秒杀系统
- 设计分布式搜索系统（Elasticsearch 集群）

**高级（加分项）**：
- 设计 AI Agent 服务（多租户，高并发）
- 设计 RAG 系统（大规模知识库）

### 系统设计答题框架

```
1. 澄清需求（2 min）：功能范围？规模（QPS/数据量）？一致性要求？
2. 规模估算（2 min）：QPS = DAU × 操作频率 / 86400；存储 = 数据量 × 记录大小
3. 高层架构（5 min）：画出核心组件图（Client/LB/Service/Cache/DB/MQ）
4. 深入细节（15 min）：重点模块的实现（数据模型/核心接口/关键算法）
5. 扩展优化（5 min）：如何扩容？如何保证高可用？瓶颈在哪？
```

## 五、模拟面试阶段（第 11-12 周）

### 每天的训练内容

```
上午（1.5h）：
  算法：1 道 Medium + 1 道 Hard，限时 40 分钟，模拟真实面试节奏
  
下午（1.5h）：
  八股：随机抽 5 个知识点，口头表达（录音回放检查流畅度）
  
晚上（1h）：
  系统设计：每两天做一道，用纸画架构图
```

### 最后 2 周查漏清单

**算法**：
- [ ] 双指针 3 种模式（对向/同向/快慢）
- [ ] 二分查找 2 种模板（左闭右闭/左闭右开）
- [ ] 单调栈（下一个更大/更小/区间问题）
- [ ] 回溯统一模板（选择/递归/撤销）
- [ ] DP 经典问题（LIS/LCS/背包/编辑距离）手写无 bug

**八股**：
- [ ] TCP 三次握手/四次挥手完整说出来
- [ ] MySQL MVCC + 间隙锁防幻读
- [ ] Redis 缓存穿透/击穿/雪崩三种解决方案
- [ ] Java HashMap 红黑树转换条件和原因
- [ ] LLM 幻觉的产生原因和工程缓解手段

## 八、心理建议

1. **量不是关键，质才是**：刷 100 道题，每题都能独立写出，强过刷 400 道、一半靠背。
2. **学一个知识点就要能讲出来**：能用自己的话解释给别人听，才是真正掌握。
3. **面试是双向选择**：准备好"反问面试官"的问题（团队氛围、技术栈方向、你认为最有挑战的问题是什么）。
4. **前两轮失败是正常的**：把每次面试当做信息收集，总结哪些知识点还不扎实。
5. **保持身体状态**：面试周期长，睡眠和运动比额外刷一道题更重要。
